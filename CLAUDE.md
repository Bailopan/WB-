# Daily AI + Wildberries News Digest — routine instructions

Этот файл — **главная инструкция для Claude Code Routine**, запускающегося каждое утро на серверах Anthropic.
Сам routine-промпт в UI — одна строка: «Выполни daily digest по инструкции в CLAUDE.md».
Всё, что ниже, — твой план действий.

## Твоя цель

За один прогон собрать свежие новости (**за последние 24 часа**) по двум темам — **AI** и **Wildberries** — из RSS-лент, перечисленных в `feeds.yaml`, написать короткий дайджест на русском и отправить его в Telegram пользователю.

## Алгоритм

1. **Прочитай `feeds.yaml`** — там две секции: `ai` и `wildberries`, каждая содержит массив объектов `{name, url}`.

2. **Запомни границу времени**: `cutoff = now - 24h`. Используй `date -u -d '24 hours ago' +'%Y-%m-%dT%H:%M:%SZ'` для получения ISO-8601.

3. **Для каждой ленты** вызови `WebFetch` с URL и промптом вида «верни список записей с полями: заголовок, ссылка, дата публикации, короткое саммари». Если лента недоступна — **не падай**, запиши ошибку в локальный список `failed_feeds` и продолжай.

4. **Отфильтруй записи**: оставь только те, у которых `published >= cutoff`. Если в ленте нет явной даты — бери топ-5 позиций (считаем что RSS отсортирован по убыванию).

5. **Дедуплицируй** — если одна и та же новость встречается в нескольких лентах (совпадение по заголовку или URL), оставь одну, указав первоисточник.

6. **Напиши дайджест** в таком формате (HTML, потому что `parse_mode=HTML`):

   ```
   <b>📰 Дайджест AI + WB — DD.MM.YYYY</b>

   <b>🤖 AI</b>
   • <a href="URL">Заголовок</a> — 1–2 предложения сути на русском.
   • <a href="URL">Заголовок</a> — …
   … (3–7 пунктов)

   <b>🛍 Wildberries</b>
   • <a href="URL">Заголовок</a> — 1–2 предложения сути на русском.
   … (3–7 пунктов)
   ```

   Правила:
   - Язык: **русский**, деловой тон, без воды, без маркетинговых клише.
   - Если заголовок на английском — переведи.
   - Эмодзи — **только** как разделители секций (🤖, 🛍, 📰). Внутри пунктов не используй.
   - Если в секции за 24ч пусто — напиши одной строкой «Существенных новостей нет.»
   - Не пересказывай факты, которых нет в источнике. Не додумывай.
   - В конце, если `failed_feeds` непуст, добавь мелким шрифтом:
     ```
     <i>⚠️ Не удалось прочитать: name1, name2</i>
     ```

7. **Отправь в Telegram**:
   ```bash
   printf '%s' "$DIGEST_TEXT" | bash scripts/send_telegram.sh
   ```
   Скрипт сам порежет на чанки если длиннее 4000 символов.

8. **Залогируй запуск**:
   - Создай ветку `claude/daily-digest-YYYY-MM-DD` (дата в UTC).
   - Запиши отправленный текст в `logs/YYYY-MM-DD.md` с префиксом-заголовком `# Дайджест от DD.MM.YYYY HH:MM UTC` и списком источников, из которых реально что-то взяли.
   - `git add logs/ && git commit -m "logs: daily digest YYYY-MM-DD" && git push -u origin claude/daily-digest-YYYY-MM-DD`
   - PR создавать **не надо**.

## Что делать при ошибках

- **Telegram API вернул не 200** — скрипт сам напишет ошибку в stderr и упадёт. Повторять не пробуй (скорее всего невалидный токен или chat_id — это нужно чинить пользователю).
- **Все ленты упали** — отправь в Telegram сообщение «⚠️ Не удалось получить ни одну RSS-ленту. Проверь feeds.yaml и network access в Cloud Environment.» и заверши работу с ошибкой.
- **Нет новостей ни по AI, ни по WB за 24ч** (редко, но бывает на выходных) — всё равно отправь сообщение с двумя строчками «Существенных новостей нет.», чтобы пользователь видел, что бот жив.

## Что трогать нельзя

- Не редактируй `feeds.yaml` — это конфиг пользователя.
- Не удаляй старые файлы из `logs/`.
- Не меняй `.claude/settings.json`.
- Не создавай `.env`-файлы и не логируй токен.

## Переменные окружения (заданы в Cloud Environment, ты их не задаёшь)

- `TELEGRAM_BOT_TOKEN` — токен бота
- `TELEGRAM_CHAT_ID` — id получателя
